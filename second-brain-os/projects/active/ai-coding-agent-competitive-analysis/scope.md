# Scope

## In

- `TMR 全无人套件` 方向相关的核心竞品，包括自动饲喂系统、自动饲喂机器人、无人自走式 TMR 整机，以及 `TMR 主机 + 数字化` 改造参考方案
- 最近 1 年内的产品演进、关键功能、工作流程、系统组成、自动化程度和商业化方案
- 各竞品的目标牧场类型、典型使用场景、部署方式、运维要求、注意事项
- 基于当前产品方案做改造的可行性判断框架，包括可复用模块、必须补齐的能力和主要风险
- 后续以官方资料、产品手册、演示内容、代理商 / 渠道资料和高信号用户反馈为主的证据收集

## Out

- 与 TMR 无人化改造无直接关系的通用农机或牧场设备
- 纯营养配方、牧场管理软件等不涉及饲喂装备 / 自动化链路的产品
- 全量穷举所有国际品牌和所有历史机型
- 超出当前公开信息能力范围的详细成本测算、详细 BOM 级拆解和实地测试结论
- 未经用户明确要求的其他项目或全局文档修改

## Assumptions

- 当前已有一版可被改造的产品方案，但其具体能力边界仍需用户补充
- 第一版以公开可验证信息和内部已知产品信息为主，不默认做实地测试或深入拆机
- 用户给出的 8 个品牌 / 方案构成第一版核心竞品名单，后续允许增删
- 首轮目标是建立正确的研究框架与比较对象，而不是直接下最终结论

## Risks

- 当前产品方案的基线信息不足，会直接影响改造可行性的判断质量
- 竞品公开资料可能更强调宣传而不是真实工况、维护成本和故障边界
- 不同品牌在不同国家 / 区域的命名、配置和商业模式可能不同，容易造成对标偏差
- 如果不先区分“整机竞品”“系统竞品”“改造参考竞品”，后续矩阵会失真

## Dependencies

- 需要用户提供当前产品方案的基本资料或口头描述，至少说明当前产品是什么、已具备哪些能力、希望补哪些无人化能力
- 下一轮需要为用户给出的 8 个竞品建立来源包，沉淀到 `context/tool_outputs/` 或 `context/research/`
- 需要明确目标客户和优先场景，例如牧场规模、地区、喂养流程和预算带
- 需要把研究维度结构化为可复用的竞品矩阵 artifact

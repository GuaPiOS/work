## 这份 artifact 要解决什么

把竞品分析项目从错误的默认方向纠正到真实的 `TMR 全无人套件` 业务问题上，确保下一轮开始收集外部资料时，不再重新讨论背景、名单、比较逻辑和可行性判断框架。

## 核心研究问题

1. 最近 1 年，`TMR 全无人套件` 方向的核心竞品有哪些，它们分别属于整机、系统还是改造参考路径？
2. 这些竞品主要覆盖哪些使用场景、目标牧场类型和部署模式？
3. 基于当前产品方案改造为 `TMR 全无人套件`，哪些能力可能复用，哪些能力必须新增？
4. 各竞品在商业化方案、交付方式、售后和注意事项上有哪些值得参考或规避的点？

## 已确认的产品映射

1. `ATX`
   是农业场景下的果园全无人导航套件，可视为现有无人化能力底座。
2. `FS200`
   是老板新定义在 `TMR` 上用的全无人导航套件，是本项目要做出的目标产品形态。
3. `FeedLogger / Nimbo X`
   是畜牧场景下已有产品，分别代表 TMR 流程数字化与推料协同能力。
4. 最终方向
   依托 `TMR` 这种产品使用场景，把现有底座和畜牧场景产品能力结合起来，做出 `FS200`。

## 研究维度 v1

1. `产品形态与系统边界`
   看它是整机、机器人、全套系统还是 `主机 + 数字化` 方案，边界覆盖到哪一步。
2. `目标客户与适用牧场场景`
   看它适合什么规模、什么布局、什么饲喂流程和地区环境。
3. `无人化程度与工作闭环`
   看它是否覆盖取料、配料、混料、送料、推料、充电 / 补能、调度和监控闭环。
4. `关键技术路径`
   看导航、感知、控制、调度、软件平台、远程管理和数据能力怎么实现。
5. `与当前产品方案的改造匹配度`
   看哪些模块可能复用，哪些模块必须新增，哪些能力差距最大。
6. `部署条件与实施复杂度`
   看场地改造、基础设施、安装调试、培训和实施周期要求。
7. `运维、可靠性与注意事项`
   看故障点、维护要求、耗材 / 易损件、售后支持和使用风险。
8. `商业化方案`
   看整机销售、系统集成、软件收费、服务包、渠道模式和售后模式。
9. `价格带与 ROI 线索`
   看公开价格、投入产出逻辑和价值主张是否可追踪。
10. `产品演进与市场信号`
    看最近 1 年发布、升级、市场活动和渠道信号。

## 内部资料带来的初步启发

1. 当前最像“可复用底座”的不是完整 TMR 无人饲喂系统，而是三块能力的组合：`FeedLogger` 的流程数字化、`Nimbo X` 的推料联动、`ATX` 的无人化感知 / 控制 / 安全栈。
2. `FS200` 已被明确为 TMR 场景下的新导航套件定义，因此后续竞品比较要优先看“导航套件如何嵌入 TMR 场景”，而不是直接把自己当成完整自动饲喂系统。
3. 真正的可行性关键不只在导航和远程控制，还在于 TMR 场景下的取料 / 配料 / 混料 / 送料闭环、牛舍 / 室内外混合环境、脏污遮挡、运维与安全要求。

## 第一版研究计划

1. 固定内部基线
   先明确当前产品方案是什么、已具备哪些能力、要补哪些无人化模块。
2. 收集证据
   为 8 个竞品收集官方资料、产品手册、演示内容、渠道资料，并把长输出落到 `context/tool_outputs/` 或 `context/research/`。
3. 搭竞品矩阵
   用上述 10 个维度建立可复用的 `comparison-matrix-v1` artifact，并补齐每个竞品的证据引用。
4. 形成可行性判断
   在竞品证据稳定后，输出基于当前产品改造的可行性报告结构、注意事项和选型建议。

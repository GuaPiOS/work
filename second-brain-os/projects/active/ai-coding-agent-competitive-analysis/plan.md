# Plan

## Stage 1

- 目标：固定 TMR 全无人套件竞品分析的背景、研究问题、时间范围、竞品名单、研究维度和本轮 canonical artifact
- 输出：`brief.md`、`scope.md`、更新后的 `project.yaml`、`current-readset.md`、`compact-state.md`、`handoff.md`
- 主 artifact：`artifacts/current/20260317_framing_tmr-unmanned-suite-competitive-analysis_research-frame-v1.md`
- 决策点：哪些品牌属于核心竞品；哪些属于改造参考组；当前产品方案的基线应该如何定义

## Stage 2

- 目标：按研究维度为核心竞品收集可追溯证据，并建立第一版竞品矩阵
- 输出：`context/tool_outputs/` 原始采集、`context/research/` 竞品笔记、比较矩阵 artifact v1
- 主 artifact：待创建的 `comparison-matrix-v1` artifact
- 决策点：是否需要引入内部产品资料进行 gap 分析；哪些维度要加权；哪些竞品分组需要调整

## Stage 3

- 目标：形成基于当前产品改造 TMR 全无人套件的可行性判断、风险清单和选型建议
- 输出：可行性报告、长报告、选型建议、必要的 memory/decision 沉淀
- 主 artifact：待创建的 `competitive-analysis-v1` artifact
- 决策点：最终输出更偏立项判断、技术路线选择，还是市场 / 竞品长期跟踪
